Disclaimer

This sandbox is version-controlled, and has been created as a result of demand, as well as in response to tickets filed for solving specific problems. There are occasionally oddities, and experiments shown that can and in some instances should be ignored. Please contact the DDS2 Support and Automation team for further information.

Notes

Sidenav Links

This sandbox does not, as of this writing, use routing, and as a side-effect of that, the sidenav Links aren't working quite right; you have to click directly on the text of the links because link values are not being supplied to the DDS component, but instead (click) events are being acted upon in this app. The links shown in "Fun Stuff" show proper link usage.

Shell Inheritance

All wrapping components inherit from helpers/dds-component.shell.ts This inheritance is pretty good, but can probably be made better, because using AfterViewInit inside one of the inherting components cancels out the parent's.

Initialization Timing

The component shell normally initializes the DDS component within the AfterViewInit life cycle stage. However if you add the init property to any of the components' instances, it will initialize AFTER that step, whenever the DDS component is found in the DOM.
